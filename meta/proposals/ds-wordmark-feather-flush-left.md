# Wordmark composition fix: feather flush-left, full POUKAI word intact

The 0.3.0 reshape (PR #19) intended a horizontal lockup with the feather isotype flush-left of the full `POUKAI` lettering. The shipped result places the feather inline **inside** the wordmark, occupying the **O** position — the rendered output reads as `P [feather] U K A I` rather than `[feather] POUKAI`. Founder rejected this on visual review.

## Symptom (rendered output verified 2026-05-14)

At `http://localhost:4321/` consuming `@poukai-inc/ui@0.3.0`, the `<Wordmark height={56} />` inside `SiteShell` renders:

```
P  [feather]  U  K  A  I
```

— the feather glyph sits between the `P` and the `U`, displacing the `O` glyph (or merged into its position). The full word `POUKAI` is no longer readable as a single word; it reads as a five-letter glyph pattern with an isotype standing in for the second letter.

Zoomed screenshot of the current render is in the session log; PNG can be regenerated by:
```
pnpm dev --host
# then GET /, screenshot top-left ~280×110 region
```

## Wanted

A clean horizontal lockup with the feather **outside** the wordmark:

```
[feather]   POUKAI
```

- Feather isotype on the **left**.
- Full six-letter `POUKAI` wordmark on the **right**, all letters intact and legible as a single word (the `O` is a normal `O` letterform, not the feather).
- POUKAI vertically centered against the feather's height on a shared optical midline.
- Comfortable optical gap between the feather and the `P` (the ~24px figure PR #19 mentioned is fine; tune to taste).

This matches the verbatim intent in PR #19's own description:
> _"Rearranges `brand/poukai-logo.svg` path geometry so the isotype sits flush-left of the POUKAI lettering on a shared baseline — replacing the previous stacked layout (isotype above letters)"_
> _"Six letter-group tx values each shifted +140 to place lettering right of the isotype with ~24 px optical gap"_

The intent was right; the implementation overlapped the feather with the wordmark instead of placing it to the left.

## Probable root cause

Educated guess for diagnosis (the DS engineer can confirm):
- The `+140` tx shift on the letter groups wasn't enough to clear the feather's right edge — the feather may be wider than 140 viewBox units, so the shifted letters still overlap it.
- Or: the **O** letter group was shifted but the feather geometry occupies the same horizontal band, so they visually coincide.
- Or: the wordmark inner SVG inadvertently dropped the `O` letter group during the rearrangement, leaving only `P` `U` `K` `A` `I` plus the feather as the new "second character."

Whatever it is, the fix is geometric: place the feather to the LEFT of all six POUKAI letters, with no overlap.

## Suggested fix

One of these:

**Option A (preferred)**: re-shift the letter `tx` values so all six glyphs (`P`, `O`, `U`, `K`, `A`, `I`) sit cleanly to the right of the feather's bounding box, with the desired optical gap. The feather's left edge stays at viewBox `x=0` or similar; the wordmark starts at `x = feather_width + 24`.

**Option B**: re-author the geometry from the canonical `brand/poukai-logo.svg` (or the original holding page's inlined SVG, which had this composition correct — see git history at `git show 9e56cdb~1:public/index.html` in `poukai-inc/pouk.ai`). The original was visually correct; the source was just stacked rather than side-by-side, so the geometry needed reshape — but the **letter integrity** was preserved.

Either option works; the constraint is **no letter of POUKAI may be displaced or replaced by the feather**.

## Validation

After landing the fix and publishing `0.3.1`:

1. `.github/workflows/ds-bump.yml` in `poukai-inc/pouk.ai` will auto-open a draft bump PR. Merge after visual check.
2. Open `http://localhost:4321/` (or the Vercel preview).
3. Confirm the top-left brand mark reads cleanly as `[feather] POUKAI` — six full letters of POUKAI, no letter overlapping or replaced by the feather.
4. The `O` in POUKAI must look like a normal `O` letterform, not the feather glyph.
5. The composition occupies the full ~2.4:1 viewBox aspect, with the feather at the left third and POUKAI filling the right two-thirds.

## Out of scope

- Don't change `Wordmark`'s prop API.
- Don't reintroduce the stacked layout.
- Don't change the default `height={64}` or `SiteShell`'s `height={56}`.
- Don't change `viewBox` aspect ratio dramatically — staying near 2.4:1 (or wider if needed to accommodate the proper gap) is fine.
